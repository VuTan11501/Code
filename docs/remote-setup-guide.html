🖥️ Remote Terminal Setup Guide

Truy cập terminal máy team từ xa qua SSH + Tailscale — thay thế chức năng Remote bị disable

📌 Cách lấy thông tin máy bạn

# Lấy IP Tailscale của máy
tailscale ip -4

# Lấy username Windows
whoami

# Lấy MAC address (cho WoL)
Get-NetAdapter | Where-Object Status -eq "Up" | Select-Object Name, MacAddress
Tailscale tạo VPN nội bộ giữa tất cả thiết bị — không cần mở port public, không cần IP tĩnh, hoạt động qua NAT/firewall.

2. Setup trên máy Server (host)

Chỉ cần làm 1 lần trên máy muốn truy cập từ xa.

1Cài Tailscale

Download & install: tailscale.com/download/windows

Đăng nhập bằng tài khoản Google/GitHub/Microsoft → thiết bị sẽ xuất hiện trong Tailscale admin.

2Cài OpenSSH Server

Mở PowerShell (Run as Administrator):

# Thử cách 1: Windows built-in
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0

# Nếu lỗi "Class not registered", dùng cách 2:
$url = "https://github.com/PowerShell/Win32-OpenSSH/releases/latest/download/OpenSSH-Win64.zip"
Invoke-WebRequest -Uri $url -OutFile "$env:TEMP\OpenSSH.zip"
Expand-Archive "$env:TEMP\OpenSSH.zip" -DestinationPath "C:\Program Files\" -Force
& "C:\Program Files\OpenSSH-Win64\install-sshd.ps1"

3Start & cấu hình SSH

# Start service + auto-start khi boot
Start-Service sshd
Set-Service -Name sshd -StartupType Automatic

# Mở firewall
New-NetFirewallRule -Name "OpenSSH-Server" -DisplayName "OpenSSH Server" `
    -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

4Đổi default shell sang PowerShell 7

# Nếu pwsh cài qua Microsoft Store (WindowsApps) → không work cho SSH
# Cài lại qua winget để có path ổn định:
winget install Microsoft.PowerShell --source winget --accept-package-agreements

# Set default shell (chọn 1 trong 2):
# Option A: PowerShell 7 (nếu cài qua MSI/winget)
Set-ItemProperty -Path "HKLM:\SOFTWARE\OpenSSH" -Name DefaultShell `
    -Value "C:\Program Files\PowerShell\7\pwsh.exe"

# Option B: PowerShell 5.1 (luôn hoạt động, gõ 'pwsh' sau khi vào)
Set-ItemProperty -Path "HKLM:\SOFTWARE\OpenSSH" -Name DefaultShell `
    -Value "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe"

# Restart SSH
Restart-Service sshd
PowerShell 7 cài qua Microsoft Store nằm trong WindowsApps — SYSTEM account không truy cập được → SSH sẽ disconnect ngay. Phải cài qua winget/MSI.

5Verify

# Test SSH vào chính mình
ssh <USERNAME>@<TAILSCALE_IP>

# Ví dụ: ssh Admin@100.64.x.x
# Nếu thành công → thấy PowerShell prompt
# Gõ 'exit' để thoát

4. Kết nối từ PC / Mac khác

Option A: Terminal SSH (đơn giản nhất)

  1. Cài Tailscale + đăng nhập cùng account
  2. Mở Terminal / PowerShell:
ssh <USERNAME>@<TAILSCALE_IP>
  1. Cài extension "Remote - SSH" trong VS Code
  2. Ctrl+Shift+P → "Remote-SSH: Connect to Host"
  3. Nhập: <USERNAME>@<TAILSCALE_IP>
  4. Chọn platform: Windows

→ Full VS Code experience trên code của máy host, bao gồm terminal, file explorer, extensions.

Option C: Windows Remote Desktop (full GUI)

Nếu cần toàn bộ desktop (GUI apps, browser, etc.):

mstsc /v:<TAILSCALE_IP>

Hoặc dùng app Microsoft Remote Desktop trên Mac/iOS.

6. Wake-on-LAN (bật máy từ xa khi Sleep)

Yêu cầu

  • Máy host đang ở trạng thái Sleep (không phải Shutdown)
  • BIOS: Enable "Wake on LAN" / "Power On By PCI-E"
  • NIC: Enable "Wake on Magic Packet" (đã verify ✅ cho Intel Wi-Fi 6E AX211)

Thông tin WoL của máy host

FieldCách lấy
MAC AddressGet-NetAdapter | Where Status -eq "Up" | Select MacAddress
Broadcast IPIP LAN của máy host (ví dụ 192.168.1.255) hoặc Tailscale IP
Port9 (mặc định)

Cách gửi WoL packet

Từ phone: dùng app WoL (search "Wake on LAN" trên App Store/Play Store), nhập MAC address trên.

Từ PC khác (cùng LAN):

# PowerShell (cài module)
Install-Module -Name WakeOnLan -Force
Send-WakeOnLan -MacAddress "<MAC_ADDRESS>"

# Hoặc Python
pip install wakeonlan
python -c "from wakeonlan import send_magic_packet; send_magic_packet('AA:BB:CC:DD:EE:FF')"
WoL qua WiFi chỉ hoạt động khi cùng mạng LAN (cùng WiFi/subnet). Nếu muốn bật từ xa qua internet, cần thiết bị trung gian luôn bật (Raspberry Pi, NAS) hoặc Smart Plug + BIOS "Restore on Power Loss".

7. Troubleshooting

❌ "Connection refused"

  • Kiểm tra sshd đang chạy: Get-Service sshd
  • Kiểm tra firewall: Get-NetFirewallRule -Name "OpenSSH*"
  • Kiểm tra Tailscale connected: tailscale status

❌ "Connection closed" ngay sau login

  • DefaultShell path không accessible → đổi về PowerShell 5.1:
Set-ItemProperty -Path "HKLM:\SOFTWARE\OpenSSH" -Name DefaultShell `
    -Value "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe"
Restart-Service sshd

❌ "Permission denied"

  • Kiểm tra username đúng (gõ whoami trên máy host để xác nhận)
  • Đảm bảo PasswordAuthentication yes trong C:\ProgramData\ssh\sshd_config
  • Restart: Restart-Service sshd

❌ "Host unreachable" / timeout

  • Tailscale chưa kết nối → mở app Tailscale, kiểm tra status
  • Máy host đang tắt/sleep → gửi WoL hoặc bật thủ công
  • Verify bằng: tailscale ping <TAILSCALE_IP>

❌ Không có autocomplete / plugin sau SSH

  • SSH mặc định vào cmd.exe hoặc PS 5.1 → gõ pwsh để chuyển PowerShell 7
  • Hoặc fix DefaultShell (xem Bước 4 ở trên)

❌ "Class not registered" khi cài OpenSSH

  • Windows Update service bị lỗi
  • Dùng cách thay thế: download ZIP từ GitHub (xem Bước 2)
