feat(ui): Light / Dark / Auto theme switcher

Adds full light theme support with three modes: Auto (follows OS),
Light, and Dark. Persisted in localStorage and synced cross-device.

- Single --tint CSS var (255,255,255 dark / 0,0,0 light) flips every
  white-overlay (42 rgba(255,255,255,X) rewrites) without per-rule edits
- New [data-theme=light] block with shadcn light palette + softer
  shadows + color-scheme:light for native form/scrollbar adaptation
- Tailwind config uses var(--*) tokens so bg-card/text-foreground/etc
  follow the theme automatically; darkMode retargeted to [data-theme=dark]
- Inline bootstrap script in <head> applies theme + theme-color meta
  BEFORE CSS loads to eliminate FOUC on first paint
- theme.js module with auto-mode matchMedia subscription so the page
  flips live with the OS
- CloudSync registers wf_dash_theme as 5th synced key; applyToUI calls
  Theme.apply FIRST before re-rendering so children paint correct colors
- Settings: new Appearance card with Auto/Light/Dark segmented buttons
- Icons: sun, moon, monitor (Lucide)

Cache: style v=46, icons v=42, theme v=1 (new), cloud-sync v=4,
       app v=33, settings v=31.
